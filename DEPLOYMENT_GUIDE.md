# StackTracker Deployment Guide

## Overview
This branch (`deployment/aws-docker-consolidated`) contains all AWS and Docker deployment files for StackTracker. This is your one-stop location for all deployment-related configurations.

## 🚀 Quick Start
This project is configured for AWS deployment using:
- **Docker** for containerization
- **AWS CDK** for infrastructure as code
- **Elastic Beanstalk** for application hosting
- **RDS PostgreSQL** for database
- **S3** for file storage

## 🐳 Docker Setup

### Main Dockerfile Configuration
The production Dockerfile uses a multi-stage build:
1. **Builder stage**: Installs dependencies and builds the application
2. **Runtime stage**: Creates lean production image

Key features:
- Node.js 20 slim base image
- Vite + esbuild for optimized builds
- Exposes port 3001
- Production-ready environment

### Building the Docker Image
```bash
# Build the Docker image
docker build -t stacktracker .

# Run locally for testing
docker run -p 3001:3001 stacktracker
```

## 🚀 Deployment Process

### Prerequisites
1. AWS CLI configured with appropriate permissions
2. AWS CDK CLI installed (`npm install -g aws-cdk`)
3. Docker installed
4. Node.js 20+ installed

### Step 1: Deploy Infrastructure
```bash
cd infra
npm install
cdk bootstrap  # Only needed once per account/region
cdk deploy StackTrackerInfraStackV3
```

### Step 2: Deploy Application
```bash
# Build and deploy via Elastic Beanstalk
npm run build
eb deploy  # If using EB CLI
# OR deploy Docker image to ECR and update EB environment
```

### Step 3: Verify Deployment
- Check Elastic Beanstalk environment health
- Verify database connectivity
- Test file upload functionality
- Monitor CloudWatch logs

## 🔐 Security Configuration

### Database Security
- Database in isolated subnet (no internet access)

### Application Security

## 🚨 Troubleshooting

### Common Issues

1. **Docker Build Fails**
   - Check Node.js version compatibility
   - Verify all dependencies are declared
   - Check Docker daemon is running

2. **CDK Deployment Fails**
   - Verify AWS credentials and permissions
   - Check CDK version compatibility
   - Review CloudFormation stack events

3. **Application Won't Start**
   - Check environment variables in EB
   - Verify database connectivity
   - Review CloudWatch logs

4. **Database Connection Issues**
   - Verify security group rules
   - Check database secret configuration
   - Test from EC2 instance in same VPC

### Useful Commands
```bash
# View CDK diff before deployment
cdk diff

# Destroy infrastructure (USE WITH CAUTION)
cdk destroy

# View EB environment logs
eb logs

# Check Docker container locally
docker logs <container_id>
```

### Recommended CI/CD Pipeline
1. **Code Push** → Trigger pipeline
2. **Tests** → Run automated tests
3. **Build** → Create Docker image
4. **Deploy Infrastructure** → CDK deploy (if changed)
5. **Deploy Application** → Update EB environment
6. **Health Checks** → Verify deployment success
7. **Rollback** → Automatic rollback on failure

### Environment Promotion
- Development → Staging → Production

- Database migrations handled separately
